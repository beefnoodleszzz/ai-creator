1. 视觉质感调优（针对 CSS/Tailwind 细节）
目标： 消除“Demo 感”，通过极细微的样式调整增加产品的“大厂工业味”。
Prompt:
“请对现有的 [组件名] 进行视觉调优。要求：
边框细节：将原有的粗边框改为极细的 0.5px 或 1px 浅色描边 border-black/5 或 border-gray-100，增加精致感。
阴影层次：去除默认阴影，改用多层弥散阴影（Soft Ambient Shadows），模拟物体漂浮在背景上的质感。
色彩升华：在背景中加入极其淡雅的径向渐变（Radial Gradient），并在卡片背后添加一层高斯模糊（Backdrop Blur）处理。
圆角规范：统一所有卡片为 rounded-3xl，所有按钮为 rounded-full，并在点击状态下增加 scale-95 的缩放反馈。
字体细节：正文字重调整为 font-light 或 font-normal，标题间距收紧 tracking-tighter，确保文字在 H5 屏幕上具有呼吸感。”

目标： 让 H5 页面操作起来像原生 App 一样流畅。
Prompt:
“请针对移动端 H5 环境，对 [交互页面名] 进行体验微调：
滚动优化：为长内容区域添加 overscroll-behavior-contain，并确保滚动条样式在 iOS/Android 上不可见但滚动平滑。
触觉反馈：在用户点击『生成』和『成功复制』时，通过 navigator.vibrate 调用轻微的物理震动反馈。
