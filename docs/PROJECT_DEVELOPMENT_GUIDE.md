# AI Creator 项目开发说明与剩余工作计划

## 1. 项目定位与目标

### 1.1 产品定位
`AI Creator` 是面向内容创作者（小红书/抖音）的一体化文案生成与运营优化工具，核心价值是：
- 快速生成可发布的高质量内容草稿
- 降低创作门槛并提升稳定产出效率
- 提供可追踪、可优化的运营数据闭环

### 1.2 目标定义（“超一流线上产品”）
以线上一流工具为标准，产品必须同时满足：
1. **可用性**：稳定、快速、移动端体验连贯
2. **质量**：输出质量可评分、可持续优化
3. **安全性**：内容风控、错误治理、审计可追踪
4. **增长能力**：账号体系、留存、转化漏斗可运营
5. **工程质量**：组件标准化、测试完备、可持续迭代

---

## 2. 当前能力（已具备）

- 核心创作链路：平台 -> 行业场景 -> 语气 -> 生成结果
- 生成稳定性：支持中断控制、失败降级、错误码标准化
- 质量体系：质量评分、风控分级（safe/warn/block）
- 数据体系：埋点、漏斗、历史、运营看板 `/ops`
- 账号体系（第一阶段）：手机号验证码登录、会话、设备历史绑定
- 数据加载优化：`useFetch` + `lazy: true` + 缓存策略

---

## 3. 与一流线上产品的关键差距（Gap）

### 3.1 产品层
- 缺少完整账号中心（资料、安全、设备管理、偏好配置）
- 缺少“可直接用于增长”的模板与任务体系（任务引导、产出目标、连续打卡）
- 缺少成体系的运营实验能力（A/B、分群、策略对比）

### 3.2 体验层
- 基础 UI 组件还未完全统一到 shadcn 体系
- 历史/看板交互密度和效率仍有提升空间（筛选、批量、检索）
- 状态反馈还可更精细（空状态、错误恢复、加载分层）

### 3.3 工程层
- 账号相关用例与回归测试覆盖不足（会话生命周期、同步冲突）
- 缺少更细粒度性能与稳定性指标（端到端耗时拆解、异常追踪）
- 规范沉淀不足（组件准入、接口契约、发布门禁）

## 5. 剩余工作计划（执行版）

## 阶段 A：设计系统与组件标准化（1-2 周）
目标：消除原生控件分散实现，建立统一可复用 UI 基座。

交付：
- shadcn 组件清单与替换矩阵
- 全页面控件替换完成（尤其表格、筛选、表单）
- 组件使用规范文档（禁止手写基础控件）

验收标准：
- 基础控件 100% 走 `app/components/ui/*`
- 页面无裸 `button/input/select/textarea`（特殊场景需注释说明）

## 阶段 B：账号体系增强（1-2 周）
目标：把“可登录”升级为“可长期留存”。

交付：
- 账号中心页：资料、安全、登录设备、最近活动
- 会话管理：过期、刷新、主动下线
- 历史同步策略可配置（账号优先/设备优先）

验收标准：
- 登录态稳定；弱网/刷新/多端场景无异常丢失
- 账号相关 e2e 覆盖关键路径

## 阶段 C：运营与增长中台（2 周）
目标：从“看数据”升级为“驱动动作”。

交付：
- `/ops` 分群筛选（游客/登录/用户ID/平台/语气/日期）
- 漏斗拆解（到步骤级）+ 异常自动标注
- 周报增强（问题定位 + 建议动作 + 影响评估）

验收标准：
- 看板可以直接支持“发现问题 -> 导出 -> 执行优化”
- 指标定义与口径统一且可追溯

## 阶段 D：质量与稳定性闭环（持续）
目标：形成持续优化机制而非一次性修补。

交付：
- 质量评分维度校准与阈值告警
- 错误码治理与故障演练手册
- 发布门禁（测试通过、指标健康、回滚预案）

验收标准：
- 连续版本迭代下，核心指标趋势稳定向好

## 6. 项目开发基调（团队约定）

### 6.1 产品基调
- **实用优先**：每个功能必须对应真实用户场景
- **可运营**：所有关键能力必须可观测、可导出、可复盘
- **低认知负担**：交互明确，状态反馈及时，不堆砌装饰

### 6.2 研发基调
- **标准化优先**：基础组件必须走 shadcn 体系，禁止重复手写
- **小步快跑**：每次改动可测试、可回滚、可评估收益
- **数据驱动**：以指标和实验结果做决策，不凭直觉拍板
- **质量红线**：主路径测试不过不发布，错误码与日志必须完备

### 6.3 设计与前端基调
- 页面是工具，不做营销页化
- 优先信息密度与操作效率，不做无意义视觉炫技
- 统一栅格、间距、字号与组件状态（hover/active/disabled/loading）

### 6.4 协作基调
- 需求必须有：目标、范围、验收标准
- PR 必须有：改动说明、风险点、测试结果
- 线上问题必须有：复盘、修复、预防三段闭环
